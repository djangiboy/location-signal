AI:
The install problem is not mainly an “address text” problem.
It is a **ground-truth place recognition** problem.

There are 3 different realities:

1. **What the user types**
2. **Where the user actually is**
3. **Where the partner can actually recognize and install**

Your system should transform 1 → 2 → 3.

Use:

* live/continuous GPS for some seconds
* nearby road/street/place labels
* nearby landmark candidates
* reverse geocode / address descriptors
* maybe compass heading + motion state if available

This helps answer:

* is user stationary?
* is GPS stable?
* what nearby landmark keeps recurring?
* what is the likely real entrance/street?

A single pin is weak.
A short stream gives a **confidence envelope**.

The system should convert noisy lat/lng into:

* nearest recognizable landmark
* lane/street/road name
* building/market/gate/society name if available
* distance + direction from landmark

Because partners often navigate by:

* “near Hanuman Mandir”
* “behind XYZ school”
* “3rd lane after petrol pump”


not by formal address strings.

So you want a **landmark-first install descriptor**.

You need a confidence model like:

* GPS stability score
* repeated pings from same place
* whether location is current or stale
* whether place lies inside plausible installable/residential zone

So yes:
**“Are you physically at the place where WiFi is to be installed?”**
should be a formal check, not an assumption.
